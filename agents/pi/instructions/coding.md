# Coding Context

You are now in a coding context. Apply these rules:

- Read the file before editing. Understand existing patterns before modifying.
- Match the existing code style — don't impose a different style.
- Make the minimal change that solves the problem. No drive-by refactors.
- Run tests after changes. If tests break, fix them before moving on.
- One logical change per commit. Don't bundle unrelated changes.

BEFORE writing code:
1. Do I understand the existing pattern?
2. Is my change the minimal fix?
3. Will existing tests still pass?
